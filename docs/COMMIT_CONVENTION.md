# 提交信息规范

本文档概述了 Talos-XII 项目中提交信息的标准格式。遵守这些准则可确保提交历史记录清晰、可读且便于自动化工具处理。

## 1. 语言要求
*   **仅限英语**：所有提交信息必须完全用英语书写。
*   **请勿使用非英语字符**：严禁使用中文、日文、韩文或任何其他非英语文字。
*   **请勿使用表情符号**：请勿在提交标题或正文中使用表情符号。

## 2. 信息结构
每条提交信息由一个**标题**和**正文**（文件列表）组成，两者之间用一个空行分隔。

```
<标题>

<正文>
```

### 2.1 标题格式
标题必须遵循以下特定模式：
```
<类型>: <摘要> in <位置>
```

*   **类型**：必须是以下之一：
    *   `feat`：新功能
    *   `fix`：错误修复
    *   `docs`：仅文档更改
    *   `style`：不影响代码含义的更改（空格、格式等）
    *   `refactor`：既不修复错误也不添加功能的代码更改
    *   `test`：添加缺失的测试或更正现有测试
    *   `chore`：构建过程或辅助工具和库的更改
*   **摘要**：对“更改了什么”的简要描述。
    *   使用祈使句、现在时（例如，“Add”、“Fix”、“Change”）。
    *   首字母必须大写。
    *   最多 50 个字符。
    *   末尾没有句号（.）。
*   **位置**：受影响的模块、组件或目录路径（例如，`src/ppo`、`config`、`README`）。

### 2.2 正文格式
正文必须列出所有修改过的文件，每行一个，相对于仓库根目录。

```
- <文件路径_1>
- <文件路径_2>
...
```

## 3. 示例

**正确示例：**
```
feat: Add fast init mode in src/config

- src/config.rs
- src/main.rs
```

**不正确示例（类型错误）：**
```
update: Update config logic in src/config

- src/config.rs
```

**不正确示例（非英语）：**
```
feat: 添加快速启动模式 in src/config

- src/config.rs
```

**不正确示例（缺少位置）：**
```
fix: Fix crash on startup

- src/main.rs
```

**不正确示例（表情符号）：**
```
feat: 🚀 Boost performance in src/worker

- src/worker.rs
```

## 4. 执行
*   **本地**：`scripts/commit-msg.sh` 中提供了一个 `commit-msg` 钩子脚本。它将在提交前验证您的信息。
*   **CI**：持续集成管道将拒绝不符合此约定的提交。
